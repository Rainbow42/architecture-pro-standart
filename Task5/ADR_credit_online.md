### **Нефункциональные и архитектурно значимые требования**

|**№**|**Требование**|
| :-: | :- |
| NFR-1 | Онлайн-заявки не идут через суточный batch АБС → Кредитный конвейер |
| NFR-2 | Скоринг не нагружается предрасчётами в рабочие часы |
| NFR-3 | Предодобренные предложения — только ночной batch |
| NFR-4 | Кредитный конвейер и скоринг — горизонтальное масштабирование, 24/7 |
| NFR-5 | HTTPS, СМС-подтверждение в ИБ (как в депозитах) |
| NFR-6 | Сквозной ID заявки между сайтом, ИБ, отделением, конвейером |
| NFR-7 | Не перегружать АБС online-запросами — CAS как прослойка |

### **Решение**

#### Архитектурное решение

Вводится **Credit Application Service (CAS)** — Java Spring Boot, по аналогии с DAS:
- принимает заявки с сайта и интернет-банка;
- вызывает **скоринг online** только в момент подачи заявки;
- передаёт заявки **напрямую в Кредитный конвейер** (REST API Camunda);
- хранит сквозной ID заявки;
- инициирует СМС через шлюз.

**Предодобренные предложения:** ночной batch-job читает данные клиентов из АБС (batch, как сейчас), вызывает скоринг в off-peak, сохраняет предложения в CAS. Днём клиент видит их в ИБ без нагрузки на скоринг.

**Отделение:** менеджер ищет заявку по ID в АБС (синхронизация CAS → АБС для карточки клиента после идентификации).

#### Диаграммы

- C4 Context: `c4_context_credit.drawio`
- C4 Component: `c4_component_credit.drawio`

#### Компоненты и интеграции

| Компонент | Интеграции |
|-----------|------------|
| Credit Application Service | ← Сайт, ИБ; → Кредитный конвейер (REST); → Скоринг (REST); → СМС |
| Кредитный конвейер (Camunda) | ← CAS; → Скоринг (REST); ↔ АБС (batch статусов, как сейчас) |
| Скоринг (Flask) | ← CAS, Конвейер; → БКИ (online); ← АБС (batch данные) |
| Интернет-банк | UI кредитов, СМС, → CAS |
| Сайт | UI кредитов, → CAS |

### **Альтернативы**

| Альтернатива | Почему не выбрана |
|--------------|-------------------|
| Заявки через АБС → batch → Конвейер | Не укладывается в SLA «решение в тот же день» |
| Предрасчёт скоринга днём для всех клиентов | Перегрузка скоринга в рабочие часы |
| Прямой вызов конвейера из ИБ | Нет единой точки для сайта, ИБ и ID заявки |
| Полная замена Camunda | Слишком дорого, конвейер уже масштабируется |

### **Недостатки, ограничения, риски**

- **Ограничение:** batch АБС ↔ Конвейер остаётся для заявок из отделения — два пути интеграции нужно поддерживать.
- **Ограничение:** новый клиент с сайта всё равно идёт в отделение для идентификации.
- **Риск:** CAS — новая система, нужна команда сопровождения.
- **Риск:** ночной batch предодобрений может устареть к моменту подачи — повторный скоринг при подаче обязателен.
- **Риск:** зависимость от доступности БКИ и скоринга для instant-решения на сайте.
